# 📦 カテゴリ 1: TypeScript 深化

> TypeScript の基礎はあるとのことなので、もう一段深い部分を確認します。

### 質問

**Q1-1.** `interface` と `type` の違いを説明してください。どちらをいつ使いますか？

```
あなたの回答:
わからない。同じだと思っていた。というか、Typeはinterfaceを兼ねるというか、typeと書いておけば通用するのかと思っていた。

```

**Q1-2.** 以下のコードのエラーを説明し、修正してください。

```typescript
function getLength(value: string | string[]) {
  return value.map(v => v.length);
}
```

```
あなたの回答:
valueがstringの場合はmap()は使えない。
function getLength(value: string | string[]) {
  if (typeof value = string){
   return value.length
  } else {
  return value.map(v => v.length);
  }
}


```

**Q1-3.** `Partial<T>`, `Pick<T, K>`, `Omit<T, K>` をそれぞれいつ使いますか？実際に使ったことはありますか？

```
あなたの回答:
正確には理解していない。
`Partial<T>`, Type T の一部分だけ。必須のものがなくてもいいが、Tに存在しないものは使えない。
`Pick<T, K>`, Type Tのうち、prop Kのtypeだけを使う
`Omit<T, K>`, Type Tのうち、prop Kをのぞいたtypeだけを使う

```

**Q1-4.** `unknown` と `any` の違いは何ですか？`any` を使うことの問題点は？

```
あなたの回答:
unknown はどのtypeかあらかじめわからないが、入ってきた型でその先に進む場合。
anyはtypescriptを使うメリットを無くしてしまう。型チェックをしないで先に進む場合。実際にはその先でエラーになってしまう場合があることが問題。


```

**Q1-5.** 以下のジェネリック関数は何をしていますか？

```typescript
function firstItem<T>(arr: T[]): T | undefined {
  return arr[0];
}
```

```
あなたの回答:
arrという配列の０番目の値を返しているが、配列がstringなのか、numberなのか、など、中身の型は何がきても入ってきた型とリターンした値の方が同じであることを指している。

```
